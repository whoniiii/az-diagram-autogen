Azure 인프라 운영의 일상적 고충

🖱️ 포탈에서 리소스 하나 만들려면 클릭 수십 번, 설정 항목은 끝이 없음
🤔 "이 리소스 그룹에 뭐가 있더라?" — 기존 인프라 구조 파악이 어려움
📝 인프라 구성을 다이어그램으로 문서화하기 번거로움 (금방 outdated)
📚 IaC(코드로 인프라 관리)를 쓰고 싶지만 학습 곡선이 높음
⚠️ 서비스 간 연결·보안·네트워크 설정에서 실수가 나기 쉬움
🔍 인프라 변경할 때 뭐가 바뀌는지 미리 확인하기 어려움

Azure Arch Builder가 하는 일

1 자연어로 요청 — "RAG 챗봇 인프라 만들어줘"
2 대화형 설계 — 서비스, SKU, 리전, 네트워크 등을 질의응답으로 결정
3 아키텍처 다이어그램 자동 생성 — 605+ Azure 아이콘 기반 인터랙티브 HTML
4 Bicep IaC 코드 자동 생성 & 배포 — 모듈별 코드 + 컴파일 검증 + 안전한 배포

주요 기능 4가지

🗣️ 대화형 인프라 설계
자연어로 요청하면 필요한 정보를 역질문하며 설계를 진행. MS Docs에서 실시간으로 서비스 가용성, SKU, API 버전을 확인
🔍 기존 리소스 스캔
az CLI로 기존 리소스를 자동 수집하고, 서비스 간 관계(PE→서비스, VNet→Subnet 등)를 추론하여 현상태 다이어그램 생성
📊 인터랙티브 다이어그램
605+ Azure 공식 아이콘, 드래그·줌·클릭 가능한 HTML 다이어그램. 설계→배포 과정에서 총 3단계 다이어그램 생성 (Draft → Preview → Result)
⚙️ Bicep 자동 생성
모듈별 구조(network, ai, storage, PE), 보안 기본값 적용, az bicep build 컴파일 검증 후 What-if 시뮬레이션으로 안전 배포

기존 방식 vs Azure Arch Builder

항목기존 방식Azure Arch Builder
인프라 설계문서 정리 + 수동 다이어그램대화로 설계 → 자동 다이어그램
IaC 작성Bicep/ARM 직접 작성자동 생성 + 컴파일 검증
서비스 설정문서 찾아가며 하나씩MS Docs 실시간 검증 자동
보안 설정항목별 수동 확인PE, RBAC, 네트워크 격리 기본 적용
인프라 문서화별도 작업 (Visio, draw.io 등)설계와 동시에 다이어그램 생성
배포 전 확인직접 검토 또는 빠뜨림What-if 시뮬레이션 + Preview 다이어그램

Phase별 상세 동작

각 Phase에서 어떤 일이 일어나는지

Phase 0 — 기존 인프라 스캔

1 Azure 로그인 확인 — az account show로 현재 로그인 상태 확인
2 구독·리소스 그룹 선택 — 사용자에게 스캔 범위를 질문
3 리소스 자동 수집 — az resource list + 서비스별 상세 쿼리 실행
4 관계 추론 — PE→서비스, VNet→Subnet, Foundry→AI Search 등 연결 관계 자동 감지
5 현상태 다이어그램 생성 — 인터랙티브 HTML 다이어그램(00_arch_current.html) 자동 생성 및 브라우저 표시

Phase 1 — 대화형 아키텍처 설계

1 프로젝트 정보 수집 — 프로젝트 이름, 필요한 서비스, SKU 등을 역질문
2 MS Docs 실시간 검증 — 선택한 리전의 서비스 가용성, 모델 가용성, SKU 옵션 확인
3 아키텍처 다이어그램 생성 — 확정된 설계를 기반으로 Draft 다이어그램(01_arch_diagram_draft.html) 생성
4 사용자 확인 — 다이어그램을 보며 수정 요청 가능, 확정 시 다음 Phase로

Phase 2-3 — Bicep 코드 생성 & 검증

1 API 버전 확인 — MS Docs에서 각 서비스의 최신 안정 버전 조회 (하드코딩 없음)
2 모듈별 Bicep 생성 — main.bicep + modules/(network, ai, storage, private-endpoints 등)
3 보안 기본값 적용 — PE, RBAC, publicNetworkAccess:Disabled, Key Vault 등 자동 설정
4 컴파일 검증 — az bicep build로 문법·구조 오류 확인, 오류 시 자동 수정 후 재컴파일

Phase 4 — 안전한 배포

1 Validate — 정책 위반, 스키마 오류 사전 확인
2 What-if 시뮬레이션 — 실제 배포 없이 어떤 리소스가 생성/수정/삭제되는지 미리 확인
3 Preview 다이어그램 — What-if 결과를 기반으로 배포 후 상태 다이어그램 생성
4 사용자 승인 후 배포 — 사용자가 확인하면 실제 배포 실행
5 결과 다이어그램 — 배포 완료 후 최종 아키텍처 다이어그램(03_arch_diagram_result.html) 생성

UC1: 생성되는 Bicep 코드


rag-chatbot/
├── main.bicep
├── main.bicepparam
└── modules/
    ├── network.bicep
    ├── ai.bicep
    ├── storage.bicep
    ├── keyvault.bicep
    └── private-endpoints.bicep
                        

// main.bicep
targetScope = 'resourceGroup'

param location string = resourceGroup().location
param projectName string = 'rag-chatbot'

module network 'modules/network.bicep' = {
  name: 'network'
  params: {
    location: location
    projectName: projectName
  }
}

module ai 'modules/ai.bicep' = {
  name: 'ai'
  params: {
    location: location
    projectName: projectName
    subnetId: network.outputs.peSubnetId
  }
}
// ... storage, keyvault, private-endpoints
